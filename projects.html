Sarcasm Detection Web Application

Repository ~ Live Site

According to a report from Buzzfeed News, "fake news about US politics accounted for 10.6 million of the 21.5 million total shares, reactions, and comments these English-language stories generated on Facebook this year." Fake and satirical news articles are often misinterpreted and spread as legitimate articles, having enormous real life ramifications. Clearly, detecting and interpreting sarcasm is a problem that even humans struggle with. This project is the result of an exploration into using Natural Language Processing Techniques to detect sarcasm and satire in inputted text. In this web application, users are able to type or paste in text into a text box, and application will classify the text instantly, in real time.


We began by building a web scraper to scrape hundreds of thousands of news articles from well known real and satirical news sites, like the Washington Post and the Onion, respectively. From there, we cleaned the scraped data to format it in a way that we could use to train and test different text classification techniques to find the best one to use in the backend of our final web application. We tested Naive Bayes and Logistical Regression techniques, using add-delta smoothing and a variety of different n-grams for each of these techniques. Additionally, we tested using a simple multilayer perceptron neural network, implemented using Tensorflow as a backend. In the end, we found that the simplest approach, the unigram Naive Bayes model, was the most accurate, performing with an accuracy of around 95% on the test data set. We integrated this model into our final web application.

  • Platform: Python and Javascript
  • Date: Fall 2016
  • Focus: Natural Language Processing

Christmas Carol Generator

Repository

To get into the holiday spirit (the Computer Science way), I built a Long Short-Term Memory (LSTM) recurrent neural network that generates new Christmas Carols in ABC notation. This LSTM uses Tensorflow as a backend. It reads in a data set of over 1600 Christmas Carols or a pretrained saved model and generates new carols character by character. It trains with a default batch size of 64 sequences, sequence length of 50 characters for 20 epochs using Adam optimization techniques with gradient clipping of 5.0 to prevent problems with exploding gradients. It uses character embedding and consists of 2 hidden layers, with a layer size of 512 neurons.

Listen to the example song below that it created. The neural network entitled this "When Christ Was Born Of Pure Marie"!

  • Platform: Python and Tensorflow
  • Date: December 2016
  • Focus: Neural Networks

C-Minor

Repository

For this project, I created an ahead-of-time compiler for a C-like pedantic language called C-Minor. It took in a file written in the C-Minor language and converted it to an equivalent x86 assembly file, and automatically compiling the x86 file to an executable file. This compiler began by scanning the input file into tokens, then parsed these tokens using the language's grammar (which I also wrote) into a parse tree, then typechecked it for proper datatypes before generating the x86 assembly code. If the compiler finds any issues that makes the C-Minor file invalid, it creates verbose error messages (one for each issue found) and kills the compiling process.

  • Platform: C, Bison, and Flex
  • Date: Fall 2015
  • Focus: Compilers

Tetris Wars

Repository

Tetris Wars allows users to battle their friends in a two player game of Tetris using separate computers. Users must try to last the longest and score the most points by clearing layers like in traditional Tetris, with the added twist of sabotauge! Players can collect powerups by clearing starred blocks and activating these powerups. Powerups can help users get a leg up on winning the battle by either making their game easier or making their opponent's game harder. Each player's screen shows both their and their opponent's boards, updated in real time.

The game was built entirely with Python. It has a completely custom-made server that the two clients (each player's game on their respective computers) can connect to for game play. The backend server and client functionality was built using the Twisted Python library. The frontend graphics and user interface was built using the PyGame Python library.

  • Platform: Python and PyGame/Twisted
  • Date: May 2016
  • Focus: Networking
